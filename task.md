- [ ] 防御コマンドのフラグ（`isGuarding: boolean`）をキャラクターに追加（効果は次タスクで）

- [ ] `isGuarding: true` のキャラクターが受けるダメージを50%にする処理を追加
- [ ] 自分のターン開始時に `isGuarding` をリセットする処理を追加
- [ ] 防御 → 敵攻撃 → ダメージ半減 → 次ターンで解除、の流れを手動確認

### 1-3. 行動順キュー（speed順）

> speed順に行動順を決める。
> この段階では1対1のまま。

- [ ] `combatState` オブジェクトを新設（`isActive`, `turnOrder`, `currentIndex`, `turnCount`）
- [ ] 戦闘開始時に `turnOrder` をspeedでソートして生成する処理を追加
- [ ] 同値のspeedはランダムで順序を決める
- [ ] `turnOrder` に従って行動を処理するループに書き換える
- [ ] enemyとplayerに`multi_action: 1` を追加
- [ ] turnOrderの生成時に`multi_action: N`の数だけそのキャラのorderを発行する（行動回数がN回になる）
- [ ] プレイヤーspeedが敵より低い場合に敵が先攻することを手動確認

### 1-4. Elonaダイス式ダメージ計算

> 現状の `Math.ceil(Math.random() * attack)` をダイス式に置き換える。

- [ ] `rollDice(count, side)` 関数を追加（`count` 個の `side` 面ダイスの合計）
- [ ] 通常攻撃のダメージ計算を仕様書1-3節のダイス式に置き換える
- [ ] 武器データに `dice_count`, `dice_side`, `fixed_damage`, `armor_pierce` を追加
- [ ] 素手（武器なし）時のデフォルト値を決める（例: `1d[attack]+0`）
- [ ] 武器を装備した場合・素手の場合それぞれダメージが出ることを手動確認
- [ ] `armor_pierce: N` のとき防御側のarmorをN点無視する処理を追加（`rollDice` のダメージ計算に組み込む）

### 1-5. 複数敵対応

- [ ] イベントデータの `enemy` フィールドを `js/data/enemies.js` に分離

- [ ] `enemy` 単体フィールドがある場合は `enemies: [enemy]` に変換

- [ ] 敵エリアのUIを複数体表示に対応（`renderEnemies(enemies)` 関数）
- [ ] 攻撃コマンド選択時にターゲット選択UIを表示（敵が2体以上の場合）
- [ ] `aoe: true` のコマンドは対象を敵全体にして個別にダメージ計算する処理を追加
- [ ] 敵が複数いる場合に戦闘が正常に進行することを手動確認

### 1-6. 状態異常の基盤

> 毒・麻痺・睡眠・燃焼の4種。付与はスキルで行うが、効果処理の基盤を先に作る。

- [ ] キャラクターに `statusEffects: []` フィールドを追加
- [ ] 自分のターン開始/終了時に `statusEffects` を処理する `processStatusEffects(character, start/end)` 関数を追加
  - 毒: 自分のターン終了時にMaxHPの5%ダメージ
  - 麻痺: 自分のターン開始時に確率で行動スキップ
  - 睡眠: 自分のターン開始時に被攻撃まで行動スキップ（被攻撃時に確率で即時解除）
  - 燃焼: 自分のターン終了時に固定ダメージ
- [ ] `remainingTurns` を毎ターン減算し、0になったら除去する処理を追加
- [ ] 防御の処理をprocessStatusEffectsに組み込む
- [ ] デバッグモードで戦闘中のみ処理できる、任意のキャラに任意の状態異常を付与するデバッグ機能を作成
